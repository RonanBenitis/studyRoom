Em aula, no curso 3, inicia-se o primeiro contato (nesta trilha de aprendizagem) com o GitHub.

Repositório: É como se fosse uma pasta no desktop, só que armazenado em nuvem (Cloud, em um servidor, como um computador, externo interconectado em diversos computadores através da internet).

=== NOTAS ===

- Não é possivel ter dois repositórios de mesmo nome.


=== ATALHOS ===

- repo.new = Ao colocar na busca de endereço do navegador, repo.new é um comando que encaminha o usuário a criação de um novo repositório (caso a sessão de login no GitHub esteja ativa).

=== BOAS PRATICAS ===

Como GitHub é acessado por diversos dispositivos e sistemas operacionais, são boas praticas:
- Criar nome de repositórios sem letras maiusculas e, preincipalmente, sem espaço.
- Utilizar, então, letra minuscula e tracinhos.

=== TERMOS ===

- Commit: É como se fosse uma foto, um retrato da versão a qual foi subida ao GitHub. Isso cria uma versão do projeto que progride conforme commis são realizados. Cada commit deixa um registro da versão que pode ser revisitado.
